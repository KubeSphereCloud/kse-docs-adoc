The **Overview** page provides the following information:

[%header,cols="1a,4a"]
|===
|Parameter |Description

|Workspace
|The workspace to which the federated project belongs.

|Creator
|The user who created the federated project.

|Resource Status
|The application resources created and the physical resources used in the federated project. Click the dropdown list in the upper right corner to select a specific cluster where the federated project resides and the time range for the data.

* **Application Resources**: The number of Deployments, StatefulSets, Persistent Volume Claims, Services, and Ingresses created in the federated project.

* **Physical Resources**: The CPU and memory usage of the federated project.

|Top 5 Resource Consumers
|The 5 workloads with the highest consumption of specific resources in the federated project. Click the dropdown list to select a specific cluster where the federated project resides and sort by CPU, memory, outbound traffic, or inbound traffic.
|===
