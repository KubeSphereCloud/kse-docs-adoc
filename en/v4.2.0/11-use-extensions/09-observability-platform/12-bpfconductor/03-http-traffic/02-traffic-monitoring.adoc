:ks_navigation: **网络 > HTTP 流量**

本节介绍如何查看单个服务或容器组的 HTTP 流量数据。


== 前提条件

* 您需要在 KubeSphere 平台具有 **platform-admin** 角色。有关更多信息，请参阅link:../../../../../05-users-and-roles/01-users/[用户]和link:../../../../../05-users-and-roles/02-platform-roles/[平台角色]。

* KubeSphere 平台需要安装并启用 **WizTelemetry 网络可观测** 扩展组件。

+

include::../../../../../_custom/WizTelemetry/enter-into-WizTelemetry.adoc[]

. 点击**流量监控**页签，然后进入**服务流量监控**或**容器组流量监控**页面。

. 在页面上方的下拉列表中选择集群、项目、服务或容器组，查看选定服务或容器组在指定时间范围内的 HTTP 流量详情。

* 在右上角点击image:/images/ks-qkcp/zh/icons/refresh-light.svg[refresh,18,18]可刷新页面信息。

// include::../../../../../../_ks_components/oper-selectTimeRange.adoc[]
* 在右上角点击image:/images/ks-qkcp/zh/icons/clock-2.svg[clock-2,18,18]可设置数据的时间范围。默认为**最近 30 分钟**。

[%header,cols="1a,4a"]
|===
|参数 |描述

|HTTP 请求数
|每秒 HTTP 请求的次数。

|HTTP 错误率
|请求失败的比例（如 0% 表示全部成功）。

|HTTP 延迟
|
* P99 延迟：指定时间范围内所有请求的 99% 分位延迟，即 99% 的请求的延迟小于或等于该数值。

* P95 延迟：指定时间范围内所有请求的 95% 分位延迟，即 95% 的请求的延迟小于或等于该数值。

* 平均延迟：指定时间范围内所有请求的平均延迟。

|HTTP 响应数据吞吐量
|服务器端每秒成功传输给客户端的 HTTP 响应数据的总量​​。

|HTTP 路由 TOP 10
|此处默认展示​​请求量最高的 10 个 HTTP 路由​端点​，以及关键性能指标（如延迟、错误率）。

您也可以在列表上方点击下拉列表，选择其他排序字段和排序方式。
|===
