All workspace members can view basic workspace information. The permissions required for other operations are as follows:

* Projects
+
--
[%header,cols="1a,4a"]
|===
|Permission |Allowed Operations

|Project Creation
|Create projects and become the administrator of the created projects.

|Project Management
|Create, edit, and delete projects within the workspace.

|Project Viewing
|View the project list and all resources within projects.
|===
--

* Application Management
+
--
[%header,cols="1a,4a"]
|===
|Permission |Allowed Operations

|Application Repository Management
|Add, edit, and delete application repositories.

|Application Repository Viewing
|View the application repository list.

|Application/Application Version/Application Instance Management
|Create application templates, edit application template information, install applications using application templates, upload application versions, delete application versions, publish application versions to the App Store, and delete application templates.

|Application/Application Version/Application Instance Viewing
|View the application template list, view application template details, and download Helm Chart archives.
|===
--

* Access Control
+
--
[%header,cols="1a,4a"]
|===
|Permission |Allowed Operations

// |Department Management
// |Set up department organizational structure, assign users to departments, and remove department members.

// |Department Viewing
// |View the department organizational structure.

// |Member Management
// |Invite users to join the workspace, modify the roles of workspace members, and remove workspace members.

|Member Viewing
|View the workspace member list and view workspace member details.

// |Role Management
// |Create workspace roles, edit workspace role information, edit workspace role permissions, and delete workspace roles.

|Role Viewing
|View the workspace role list and view workspace role details.
|===
