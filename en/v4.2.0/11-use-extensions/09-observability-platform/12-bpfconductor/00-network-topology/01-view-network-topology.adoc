:ks_navigation: **网络 > 网络拓扑**


本节介绍如何查看网络拓扑图和网络流量列表。


== 前提条件

* 您需要在 KubeSphere 平台具有 **platform-admin** 角色。有关更多信息，请参阅link:../../../../../05-users-and-roles/01-users/[用户]和link:../../../../../05-users-and-roles/02-platform-roles/[平台角色]。

* KubeSphere 平台需要安装并启用 **WizTelemetry 网络可观测** 扩展组件。

. 在页面上方的下拉列表中选择集群、源项目、目标项目，查看特定集群中特定项目之间的网络拓扑图和网络流量数据。
+
--
[%header,cols="1a,4a"]
|===
|参数 |描述

|源项目
|数据发送方所在的项目，也就是数据流的起点。

|目标项目
|数据接收方所在的项目，也就是数据流的终点。
|===

// include::../../../../../../_ks_components/oper-refreshListData.adoc[]
* 在右上角点击image:/images/ks-qkcp/zh/icons/refresh-light.svg[refresh,18,18]可刷新页面信息。

// include::../../../../../../_ks_components/oper-selectTimeRange.adoc[]
* 在右上角点击image:/images/ks-qkcp/zh/icons/clock-2.svg[clock-2,18,18]可设置数据的时间范围。默认为**最近 30 分钟**。
--

. **网络拓扑图**展示了各个拓扑节点的资源类型（容器组、服务、集群节点或 IP 地址）、资源名称和总传输速率，以及拓扑节点之间的交互关系。您还可通过以下操作获取更多信息。
+
--
* 将鼠标悬停在拓扑节点上，查看与该拓扑节点通信的相关节点以及数据流走向。

* 点击拓扑节点，查看该拓扑节点的资源名称、所属项目、发送速率和接收速率。

* 将鼠标悬停在拓扑连线上，查看连线两端的拓扑节点，并且可通过image:/images/ks-qkcp/zh/icons/left-right-button.svg[left-right-button,18,18]了解数据流走向。

* 点击拓扑连线或其上的image:/images/ks-qkcp/zh/icons/left-right-button.svg[left-right-button,18,18]，查看该段数据流的起点、终点，以及数据传输速率。

[%header,cols="1a,4a"]
|===
|参数 |描述

|源端
|发起通信请求的网络端点，显示该拓扑节点的资源类型和资源名称。

|目标端
|接收通信请求的网络端点，显示该拓扑节点的资源类型和资源名称。
	
|发送速率
|源端向目标端传输数据的速率。

|接收速率
|目标端从源端接收数据的速率。

|总速率
|发送速率和接收速率的总和。

|TCP 丢弃
|每秒钟 TCP 丢弃的数据包数量。

高丢弃率可能表明网络不稳定或设备过载，需优化网络配置或扩容资源。

|TCP 重传
|每秒钟 TCP 重传的数据包数量。

频繁重传通常意味着网络延迟或丢包，可能影响传输效率，需检查网络延迟或拥塞情况。
|===
