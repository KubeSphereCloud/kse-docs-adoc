:ks_menu: **Cluster Management**
:ks_permission: **Cluster Viewing**

This section describes how to view the cluster list.

== Prerequisites

You need to have the pass:a,q[{ks_permission}] permission on the KubeSphere platform.

== Steps

include::../../../../_custom/platformManagement/platformManagement-oper-logIn.adoc[]

. Click **Cluster Management**.

====
* The cluster list provides the following information:
+
--
[%header,cols="1a,4a"]
|===
|Parameter |Description

|Name, Alias, and Description
|The name, alias, and description of the cluster.

|Status
|The current status of the cluster.

|Number of Nodes
|The number of server nodes in the cluster.

|Provider
|The provider of the cluster environment.

|KubeSphere Version
|The KubeSphere version running in the cluster.

|Labels
|Labels for the cluster, used for categorizing and managing clusters.

|Creation Time
|The time when the cluster was added to the KubeSphere web console.
|===
--

* Click the search box above the list and enter a keyword to search for clusters by name.

include::../../../../../_ks_components/oper-refreshListData.adoc[]

include::../../../../../_ks_components/oper-customizeColumns.adoc[]

* Click the name of a cluster to enter the cluster and manage resources within it.

====
