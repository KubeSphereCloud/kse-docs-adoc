. On the **Resource Status** tab of the application details page, view the services, workloads, Ingresses, and persistent volume claims that make up the application.

* The **Services** area provides the following information:
+
--
[%header,cols="1a,4a"]
|===
|Parameter |Description

|Name
|The name of the service.

|Project
|The project to which the service belongs.

|Internal Access
|The virtual IP address for accessing the service from within the cluster.

|External Access
|The port number for accessing the service from outside the cluster.

// |Internal Access Mode
// |
// include::../../clusterManagement/services/services-para-internalAccess.adoc[]

// include::../../clusterManagement/services/services-para-virtualip-headless.adoc[]

// |Application Governance
// |Whether application governance is enabled for the application. After application governance is enabled, you can use the traffic monitoring, grayscale release, and link tracing features provided by KubeSphere.

// |Virtual IP Address
// |The virtual IP address for accessing the service from within the cluster. Displayed only when the internal access type of the service is **VirtualIP**.
|===
--

* The **Workloads** area provides the following information:
+
--
[%header,cols="1a,4a"]
|===
|Parameter |Description

|Name
|The name of the workload.

|Project
|The project to which the workload belongs.

// |Type
// |The type of the workload.

|Status
|The current status of the workload.

include::../../clusterManagement/workloads/workloads-para-workloadStatus.adoc[]

// |Modification Record
// |The current modification record of the workload.
|===
--

* The **Ingresses** area provides the following information:
+
--
[%header,cols="1a,4a"]
|===
|Parameter |Description

|Name
|The name of the Ingress.

|Domain
|The domain name of the Ingress.

|URL
|The access address of the service corresponding to the Ingress.
|===

Click **Access Service** on the right side of the Ingress to access the backend service of the Ingress.
--
