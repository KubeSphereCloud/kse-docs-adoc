. On the pod details page, view the resource attributes of the pod in the **Properties** section.
+
--
[%header,cols="1a,4a"]
|===
|Parameter |Description

|Cluster
|The cluster to which the pod belongs.

|Project
|The project to which the pod belongs.

|Application
|The application to which the pod belongs.

|Status
|The status of the pod.

include::pods-para-podStatus_overview.adoc[]

include::pods-para-podIpPool.adoc[]

|Node Name
|The name of the node where the pod is located.

|Node IP Address
|The IP address of the node where the pod is located.

|Restart Count
|The restart count of the pod.

|QoS Type
|The Quality of Service type of the pod, determined by the user-set memory limit, memory request, CPU limit, and CPU request of the containers.

include::pods-para-qosTypes.adoc[]

|Creation Time
|The creation time of the pod.

|Creator
|The creator of the pod.

. On the pod details page, view the containers and volumes in the pod on the **Resource Status** tab.
+
--
[%header,cols="1a,4a"]
|===
|Parameter |Description

|Containers
|Displays the name, status, restart count, and container port of the containers running in the pod. Container status types include:

include::../workloads/workloads-para-containerStatus.adoc[]

Click image:/images/ks-qkcp/zh/icons/log.svg[log,18,18] to the right of the container name to view container logs. Hover over **Probe** to view the probes configured on the container.

|Volumes
|Displays the names of all volumes in the pod, the containers to which volumes are mounted, and the mount paths of the volumes on the containers.
|===
--

. On the pod details page, click the **Scheduling Information** tab to view the scheduling results and status information of the pod.
+
--
[%header,cols="1a,4a"]
|===
|Parameter |Description

|Scheduling Result
|Displays the name of the node that received the pod.

|Status Information
|Indicates whether the pod has successfully completed all startup steps. If image:/images/ks-qkcp/zh/icons/success.svg[success,18,18] is displayed, it indicates that the pod has completed the startup steps. If image:/images/ks-qkcp/zh/icons/error.svg[error,18,18] is displayed, it indicates that the startup steps have not yet been completed.

* **Running**: Whether the current status of the pod is **Running**.

* **Pod Scheduled**: Whether the pod has been scheduled to a node.

* **Pod Initialized**: Whether all init containers in the pod have started successfully.

* **Containers Ready**: Whether all containers in the pod have started successfully.

* **Pod Ready**: Whether the pod is ready.
|===
