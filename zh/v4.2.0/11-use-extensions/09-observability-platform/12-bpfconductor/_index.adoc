本节介绍如何使用“WizTelemetry 网络可观测”扩展组件。

该组件用于在 Kubernetes 集群中部署和管理 eBPF 监控代理。内置了多个网络可观测性相关的 eBPF 程序/插件，可提供对第 4 层网络流量、第 7 层 HTTP 流量以及网络拓扑的深入洞察。

安装“WizTelemetry 网络可观测”扩展组件后，组件坞中的 WizTelemetry 可观测平台将显示**网络**菜单。您可以使用网络可观测相关的功能，包括：网络拓扑、服务拓扑、HTTP 流量。

[.admon.attention,cols="a"]
|===
|注意

|WizTelemetry 网络可观测依赖 WizTelemetry 平台服务、 WizTelemetry 监控和 WizTelemetry 数据流水线，因此在安装 WizTelemetry 网络可观测之前必须先安装并配置好依赖的扩展组件，否则网络可观测相关功能无法正常使用！
|===

== 系统要求

=== 内核版本要求

* 推荐：Linux 内核版本 5.8+ 以获取完整的 eBPF 功能支持
* 最低要求：Linux 内核版本 4.18+

=== 内核功能支持

* 确保内核已启用 eBPF 支持（CONFIG_BPF=y）
* 对于较旧的内核（4.x），强烈建议使用已回移植（backported）eBPF 特性的定制内核
** 某些发行版（如 CentOS、RHEL）可能已将 eBPF 特性回移植到旧版内核
** 请查阅您的发行版文档以确认 eBPF 功能的支持情况

=== 节点资源要求

* 确保节点有足够的资源（CPU、内存）运行 BPF 代理
* 在生产环境中为 BPF 程序预留适当的系统资源
