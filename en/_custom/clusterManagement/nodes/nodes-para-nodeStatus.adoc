The running status of each node.

* **Running**: The node is running normally.

* **Unschedulable**: The node is manually set to unschedulable by the user. If a node is in this state, pods cannot be scheduled to this node.

* **Alerting**: There are alerts on the node. You can view the alert types for the node in the **Running Status > Health Status** area on the node details page.
