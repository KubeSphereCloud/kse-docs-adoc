:ks_navigation: **网络 > 服务拓扑**


本节介绍如何实时观测集群内服务的性能。

== 前提条件

* 您需要在 KubeSphere 平台具有 **platform-admin** 角色。有关更多信息，请参阅link:../../../../../05-users-and-roles/01-users/[用户]和link:../../../../../05-users-and-roles/02-platform-roles/[平台角色]。

* KubeSphere 平台需要安装并启用 **WizTelemetry 网络可观测** 扩展组件。

== 操作步骤

include::../../../../../_custom/platformManagement/extensionManagement/extensionManagement-oper-logInAsPlatformAdmin.adoc[]

+

include::../../../../../_custom/WizTelemetry/enter-into-WizTelemetry.adoc[]

include::../../../../../../_ks_components/oper-navigate.adoc[]

. 点击**服务监控**页签，进入服务性能监控页面。

. 在页面上方的下拉列表中选择集群、项目，查看选定项目在指定时间范围内的服务性能。

--
// include::../../../../../../_ks_components/oper-refreshListData.adoc[]
* 在右上角点击image:/images/ks-qkcp/zh/icons/refresh-light.svg[refresh,18,18]可刷新页面信息。

// include::../../../../../../_ks_components/oper-selectTimeRange.adoc[]
* 在右上角点击image:/images/ks-qkcp/zh/icons/clock-2.svg[clock-2,18,18]可设置数据的时间范围。默认为**最近 30 分钟**。

* 点击**收起服务列/展开服务列**，收起或展开选定项目中的服务及其容器组的列表。点击服务或容器组，查看服务或容器组的流量监控详情。有关更多信息，请参阅 link:../../03-http-traffic/02-traffic-monitoring/[HTTP 流量监控]。

[%header,cols="1a,4a"]
|===
|参数 |描述

|HTTP 请求数
|每秒 HTTP 请求的次数。

|HTTP 错误率
|请求失败的比例（如 0% 表示全部成功）。

|HTTP P99 延迟
|指定时间范围内所有请求的 99% 分位延迟，即 99% 的请求的延迟小于或等于该数值。

|HTTP P95 延迟
|指定时间范围内所有请求的 95% 分位延迟，即 95% 的请求的延迟小于或等于该数值。

|入站吞吐量
|服务器端每秒接收的数据量。

|出站吞吐量
|服务器端每秒返回的数据量。
|===
