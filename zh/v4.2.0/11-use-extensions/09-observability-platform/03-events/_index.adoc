本节介绍如何使用“WizTelemetry 事件”扩展组件。

该组件可长期保存工作负载、任务、容器组、服务、应用路由等资源的事件，并提供多租户视角的事件检索和查看功能。

安装“WizTelemetry 事件”扩展组件后，

* 组件坞中的 WizTelemetry 可观测平台将显示**日志**菜单，**日志**菜单下将显示**事件查询**选项。

* 集群左侧导航栏将显示**日志**菜单，**日志**菜单下将显示**日志接收器**和**事件查询**选项。**日志接收器**页面将显示**资源事件**页签，支持添加多种类型的日志接收器。有关更多信息，请参阅link:../02-logging/02-log-receivers/[日志接收器]。**事件查询**页面支持用户查询自身权限范围内的资源事件。

| 注意

|
WizTelemetry 事件依赖 WizTelemetry 平台服务和 WizTelemetry 数据流水线，因此在安装 WizTelemetry 事件之前必须先安装并配置好 WizTelemetry 平台服务和 WizTelemetry 数据流水线两个扩展组件，否则事件相关功能无法正常使用！
