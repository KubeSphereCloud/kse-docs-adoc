[%header,cols="1,5"]
|===
|Parameter |Description

|Username
a|Custom username.

Configuration requirements: The username can only contain lowercase letters, numbers, and underscores (_). It must start with a lowercase letter and be 2 to 26 characters in length.

|Password
a|Enter the user password.

Configuration requirements: Can only contain uppercase and lowercase letters (A～Z, a～z), numbers (0～9), special characters (!@#$%^&*_+-=). Length must be between 8 and 32 characters.

|Authorized Database
a|Enter the name of the database authorized for access.

Default is *, which means all databases are authorized. Only one database name can be entered.

|User Permissions
|Enter the user account permissions. You can choose between Normal Permissions or Advanced Permissions.

After the application is created successfully, the default user **root** will be generated. Multiple Advanced Permissions user accounts and Normal Permissions user accounts are supported.

**root** is the super administrator user, possessing the highest management permissions for the database. This user account is generated by default and cannot be modified or deleted.

**Normal Permissions** users have database operation and usage permissions, including ALTER, ALTER ROUTINE, CREATE, CREATE ROUTINE, CREATE TEMPORARY TABLES, CREATE VIEW, DELETE, DROP, EXECUTE, EVENT, INDEX, INSERT, LOCK TABLES, PROCESS, RELOAD, SELECT, SHOW DATABASES, SHOW VIEW, UPDATE, TRIGGER, REFERENCES, and other database operation permissions. This type of user account can be added, modified, and deleted.

**Advanced Permissions** users have advanced database operation and usage permissions. In addition to Normal Permissions, they also possess SUPER and GRANT database operation permissions. This type of user account can be added, modified, and deleted.

|Authorized Host
a|Enter the host IP(s) allowed to access.

Default is %, which means any host can access. You can enter one or more host IPs, separated by commas (,).

|===
