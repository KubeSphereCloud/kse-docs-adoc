This section describes how to create a user.

== Prerequisites

You need to have the pass:a,q[{ks_permission}] permission on the KubeSphere platform. For more information, refer to link:../../01-users/[Users] and link:../../02-platform-roles/[Platform Roles].


== Steps

include::../../../_custom/platformManagement/platformManagement-oper-logIn.adoc[]
+
. Click pass:a,q[{ks_menu}].
. In the left navigation pane, select **Users**.
. On the page, click **Create**.
. In the **Create User** dialog box, set the user's information, then click **OK**.
+
[%header,cols="1a,4a"]
|===
|Parameter |Description

|Username
|The name of the user. The username can only contain lowercase letters, numbers, hyphens (-), and periods (.). It must start and end with a lowercase letter or number, with a maximum length of 32 characters.

|Alias
|The alias of the user.
// The alias can contain any characters, with a maximum length of 63 characters.

|Email
|The email address of the user. The email address can be used to log in to the KubeSphere web console.

|Platform Role
|The role of the user on the KubeSphere platform. The platform role determines the user's permissions in KubeSphere.

|Password
|The password of the user. The password must contain numbers, uppercase letters, lowercase letters, and special characters, with a length between 8 and 64 characters.

|Description
|The description of the user. The description can contain any characters, with a maximum length of 256 characters.
|===
+
After creation, the user will appear in the user list.
