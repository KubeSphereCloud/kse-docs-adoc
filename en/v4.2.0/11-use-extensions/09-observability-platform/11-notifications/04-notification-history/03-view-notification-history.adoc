This section describes how to view notification history records.

== Prerequisites

* You need to have the **platform-admin** role in the KubeSphere platform. For more information, refer to link:../../../../../05-users-and-roles/01-users/[Users] and link:../../../../../05-users-and-roles/02-platform-roles/[Platform Roles].

* The **WizTelemetry Notification** extension needs to be installed and enabled on the KubeSphere platform.

* The **Notification History** page provides the following information:
+
--
[%header,cols="1a,4a"]
|===
|Parameter |Description

|Alert Name
|The name of the alert sent as a notification to users.

|Alert Type
|The type of system event indicated by the alert.

* **Metric Alert**: Resource metrics meet the alert trigger conditions.

* **Audit Alert**: System audit events, such as API calls.

* **Event Alert**: System resource events, such as resource creation, deletion, modification, etc.

|Alert Level
|
include::../../../../../_custom/clusterManagement/ruleGroups/ruleGroups-para-alertLevel.adoc[]

|Message
|The content of the alert message.

|Status
|The status of the alert: Firing/Resolved.

**Firing**: A triggered alert generates a firing notification.

**Resolved**: If the same alert is not received again after a period of time (default 5 minutes), the status changes to Resolved.

|Project
|The project where the resource that triggered the alert is located.

|Service
|The service that triggered the alert.

|Pod
|The pod that triggered the alert.

|Container
|The container that triggered the alert.

|Notification Time
|The time the notification was sent.
|===
--

* Select a cluster from the dropdown list in the upper left corner of the list to view notification history in a specific cluster.

* Click the search box above the list and enter keywords to search notification history by alert name, alert level, message, status, project, and time range.
