本节介绍如何同步项目中的资源文件到代码仓库。同步完成后，您可以在代码仓库中管理项目的资源文件。


== 前提条件

* KubeSphere 平台需要安装并启用 **DevOps** 扩展组件。

* 已创建一个企业空间、一个 DevOps 项目和一个用户（例如 **project-regular**），并已邀请该用户至 DevOps 项目且授予 **operator** 角色。请参阅link:../../02-devops-project-mangement/03-project-members/01-invite-a-user-to-a-project/[邀请用户加入 DevOps 项目]。

* 已link:../../06-code-repositories/01-import-code-repositories/[导入代码仓库]。

. 点击**自定义资源清单**按钮，在目录树或右侧列表中，勾选待同步到代码仓库的资源文件。也可在文件内容详情页点击**加入清单**。选中的文件将显示在右侧的待同步资源清单下。

. 在待同步资源清单下，点击**同步至代码仓库**。

. 在**基本信息**页签，确认待同步的资源，然后点击**下一步**。
+
点击image:/images/ks-qkcp/zh/icons/pen-light.svg[pen-light,18,18]可编辑资源文件。编辑完成后，点击最下方的image:/images/ks-qkcp/zh/icons/check-dark.svg[check-dark,18,18]保存。

. 在**应用设置**页签，选择集群、DevOps 项目、代码仓库、分支，设置存放目录和 commit 等信息。点击**确定**开始同步。
