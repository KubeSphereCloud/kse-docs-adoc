Users with the **platform-admin** role can enable and disable extensions. The permissions required for other platform operations are as follows:

* Access Control
+
--
[%header,cols="1a,4a"]
|===
|Permission |Allowed Operations

|Workspace Viewing
|View the workspace list.

|Workspace Management
|Create a workspace, edit workspace information, delete a workspace.

|User Viewing
|View the user list, view user details.

|User Management
|Create a user, edit user information, enable and disable users, modify user roles, delete users.

|Role Viewing
|View the role list, view role details.

|Role Management
|Create platform roles, edit platform role information, edit platform role permissions, delete platform roles.
|===
--

* Cluster Management
+
--
[%header,cols="1a,4a"]
|===
|Permission |Allowed Operations

|Cluster Viewing
|View the cluster list.

|Cluster Management
|Add member clusters, edit cluster information, update kubeconfig, remove member clusters.
|===
--

* Application Management
+
--
[%header,cols="1a,4a"]
|===
|Permission |Allowed Operations

|Application Viewing
|View the application template list, view application template details, view application review records.

|Application Template Management
|Take down and publish applications, take down and publish application versions, manage application categories, set the category an application belongs to, review application versions.
|===
--

* Platform Settings Management
+
--
[%header,cols="1a,4a"]
|===
|Permission |Allowed Operations

|Platform Settings Management
|Set up email notifications, set up Feishu notifications, set up DingTalk notifications, set up WeCom notifications, set up Slack notifications, set up Webhook notifications.
|===
--
